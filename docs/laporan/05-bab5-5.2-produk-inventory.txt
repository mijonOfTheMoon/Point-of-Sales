5.2 Produk dan Inventory

5.2.1 Deskripsi Fitur

Fitur produk dan inventory digunakan untuk mengelola barang yang dijual toko beserta stoknya. Lewat fitur ini pengguna bisa menambah produk baru, mengubah harga dan stok, sampai menonaktifkan produk yang sudah tidak dijual. Data stok di sini menjadi penting karena nantinya akan dipakai modul penjualan untuk membatasi jumlah barang yang boleh masuk keranjang.

5.2.4 Kode ViewModel Produk

Kotlin

class ProductViewModel(private val repository: ProductRepository = ProductRepository()) : ViewModel() {

    private val _uiState = MutableStateFlow<ProductUiState>(ProductUiState.Loading)
    val uiState: StateFlow<ProductUiState> = _uiState.asStateFlow()

    init {
        loadProducts()
    }

    fun loadProducts() {
        viewModelScope.launch {
            _uiState.value = ProductUiState.Loading
            try {
                val products = repository.getProducts().filter { it.is_active }
                _uiState.value = ProductUiState.Success(products)
            } catch (e: Exception) {
                _uiState.value = ProductUiState.Error(e.message ?: "Failed to load products")
            }
        }
    }

...

    fun addProduct(name: String, price: Double, stock: Double) {
        viewModelScope.launch {
            try {
                repository.addProduct(Product(name = name, price = price, stock = stock))
                loadProducts()
            } catch (e: Exception) {
                _uiState.value = ProductUiState.Error(e.message ?: "Failed to add product")
            }
        }
    }

...

}

5.2.5 Penjelasan Penerapan ViewModel Produk

ProductViewModel menyimpan state daftar produk lewat _uiState yang bertipe MutableStateFlow. Seperti ViewModel lain, _uiState dibuat private supaya hanya ViewModel yang bisa mengubahnya, lalu ke luar hanya uiState yang diekspos lewat asStateFlow yang read only. Begitu ViewModel dibuat, blok init langsung memanggil loadProducts supaya daftar produk segera diambil tanpa menunggu aksi pengguna.

Ada satu hal yang membedakan modul ini, yaitu penyaringan di dalam loadProducts. Setelah mengambil produk dari repository, ViewModel tidak langsung menampilkan semuanya. Ia memakai filter dengan syarat is_active, sehingga hanya produk yang masih aktif yang disimpan ke dalam state Success. Keputusan ini diambil karena penonaktifan produk di aplikasi ini bersifat soft delete. Produk yang dihapus sebenarnya hanya ditandai tidak aktif di database, bukan dibuang, jadi penyaringan di sisi ViewModel inilah yang memastikan produk nonaktif tidak ikut muncul di layar.

Di sinilah terlihat perbedaan modul produk dari modul kas dan pengeluaran. Penambahan dan pengubahan produk di repository memakai insert serta update tabel secara langsung, bukan lewat RPC, karena operasinya memang sederhana dan tidak menyentuh perhitungan saldo. Hanya penonaktifannya yang lewat RPC deactivate_product supaya penandaan soft delete-nya terjaga di sisi database. Dengan addProduct sebagai contoh, alur penambahan produk berjalan seperti berikut:
1. addProduct menjalankan coroutine lewat viewModelScope.launch, lalu membungkus name, price, dan stock dari input pengguna menjadi sebuah objek Product.
2. addProduct memanggil repository.addProduct yang menjalankan insert objek Product tadi ke tabel product lewat Postgrest.
3. Begitu insert berhasil, addProduct memanggil loadProducts yang membaca ulang tabel product lewat repository.getProducts, menyaring hasilnya dengan filter is_active, lalu menulis ProductUiState.Success berisi daftar terbaru ke _uiState.

Karena _uiState bersifat observable, perubahan nilai pada langkah terakhir membuat ProductScreen yang mengamatinya lewat collectAsStateWithLifecycle langsung menggambar ulang daftarnya dengan data terbaru.
