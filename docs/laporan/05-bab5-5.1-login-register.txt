5.1 Login dan Register

5.1.1 Deskripsi Fitur

Fitur login dan register digunakan untuk menangani autentikasi pengguna lewat Supabase Auth. Lewat fitur ini pengguna baru bisa mendaftar dengan nama, email, dan password, lalu pengguna lama bisa masuk memakai email dan password-nya. Selain itu fitur ini juga bertugas memeriksa sesi yang masih aktif, sehingga pengguna yang sudah login sebelumnya akan langsung diarahkan ke dashboard tanpa perlu masuk ulang.

5.1.4 Kode ViewModel Auth

Kotlin

class AuthViewModel(private val repository: AuthRepository = AuthRepository()) : ViewModel() {

    private val _uiState = MutableStateFlow<AuthUiState>(AuthUiState.Idle)
    val uiState: StateFlow<AuthUiState> = _uiState.asStateFlow()

    private val _checkState = MutableStateFlow<AuthCheckState>(AuthCheckState.Loading)
    val checkState: StateFlow<AuthCheckState> = _checkState.asStateFlow()

    init {
        checkSession()
    }

    private fun checkSession() {
        viewModelScope.launch {
            if (repository.isUserLoggedIn()) {
                refreshAuthenticatedState()
            } else {
                _checkState.value = AuthCheckState.Unauthenticated
            }
        }
    }

...

    fun signIn(email: String, password: String) {
        viewModelScope.launch {
            _uiState.value = AuthUiState.Loading
            try {
                repository.signIn(email, password)
                refreshAuthenticatedState()
                _uiState.value = AuthUiState.Success("Login successful!")
            } catch (e: Exception) {
                _uiState.value = AuthUiState.Error(e.message ?: "Login failed")
            }
        }
    }

...

}

5.1.5 Penjelasan Penerapan ViewModel Auth

AuthViewModel memegang dua aliran state sekaligus, dan keduanya punya peran yang berbeda supaya tidak saling mengganggu. State pertama adalah uiState yang dipakai untuk menandai proses login atau register sedang berjalan, berhasil, atau gagal. State kedua adalah checkState yang menyimpan status sesi pengguna, mulai dari Loading, Authenticated, sampai Unauthenticated. Keduanya sama sama disimpan lewat MutableStateFlow yang private, lalu diekspos ke luar lewat asStateFlow yang read only, sehingga layar hanya bisa mengamati dan tidak bisa mengubahnya. Pemisahan dua state ini penting, karena status sesi perlu bertahan lintas layar, sementara status proses login cukup bersifat sementara.

Begitu ViewModel dibuat, blok init akan langsung memanggil checkSession supaya aplikasi tahu apakah ada sesi yang masih tersimpan dari pemakaian sebelumnya. Di dalam checkSession, ViewModel bertanya ke repository lewat isUserLoggedIn. Function itu di sisi repository menunggu status sesi dari Supabase Auth sampai jelas, lalu mengembalikan true atau false. Hasilnya menentukan arah berikutnya. Apabila pengguna memang sudah login, ViewModel memanggil refreshAuthenticatedState untuk mengambil profil terbaru lalu mengisi checkState menjadi Authenticated. Apabila belum, checkState diisi Unauthenticated. Nilai checkState inilah yang nantinya dibaca AppNavigation untuk memutuskan layar awalnya dashboard atau login.

Alur saat pengguna menekan tombol sign in bisa diurutkan seperti berikut:
1. signIn lebih dulu mengubah _uiState menjadi AuthUiState.Loading supaya tombolnya bisa menampilkan indikator dan mencegah penekanan ganda.
2. signIn memanggil repository.signIn dengan email dan password, yang lalu menjalankan signInWith(Email) ke Supabase Auth.
3. Bila kredensialnya benar, signIn memanggil refreshAuthenticatedState yang membaca profil lewat getCurrentUserProfile lalu mengisi _checkState dengan AuthCheckState.Authenticated.
4. Setelah itu signIn mengisi _uiState dengan AuthUiState.Success, dan LoginScreen yang mengamati uiState akan bereaksi terhadap keberhasilan tersebut.
5. Bila gagal, signIn menangkap exception-nya lalu mengisi _uiState dengan AuthUiState.Error berisi pesannya.

Di sisi layar, LoginScreen mengambil uiState lewat collectAsStateWithLifecycle. Begitu uiState berubah menjadi Loading, tombol sign in berganti menjadi indikator memuat. Begitu uiState menjadi Success, layar menandai login berhasil lalu memanggil resetUiState supaya status proses kembali ke Idle. RegisterScreen bekerja dengan pola serupa lewat signUp, hanya saja setelah berhasil ia memberi jeda singkat lalu mengarahkan pengguna kembali ke halaman login.
