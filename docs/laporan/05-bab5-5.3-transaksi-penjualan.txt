5.3 Transaksi Penjualan

5.3.1 Deskripsi Fitur

Fitur transaksi penjualan menjadi inti dari aplikasi ini karena di sinilah proses jual beli dijalankan. Pengguna akan memilih produk, mengatur jumlahnya di keranjang, lalu masuk ke tahap checkout untuk menentukan kas penerima dan pelanggannya. Saat pembayaran diproses, transaksi tersebut akan langsung tersimpan sekaligus mengurangi stok produk dan menambah saldo kas yang dipilih. Seluruh riwayat transaksinya nantinya bisa ditinjau ulang lewat halaman terpisah.

5.3.4 Kode ViewModel Penjualan

Kotlin

fun addToCart(product: Product, quantity: Double = 1.0) {
    val productId = product.id ?: return
    if (quantity == 0.0) return
    val maxQuantity = product.stock.coerceAtLeast(0.0)
    if (quantity > 0.0 && maxQuantity <= 0.0) return

    val currentCart = _cart.value.toMutableList()
    val existingItemIndex = currentCart.indexOfFirst { it.product.id == productId }
    if (existingItemIndex != -1) {
        val existingItem = currentCart[existingItemIndex]
        val updatedQuantity = (existingItem.quantity + quantity).coerceAtMost(maxQuantity)
        if (updatedQuantity > 0.0) {
            currentCart[existingItemIndex] = existingItem.copy(quantity = updatedQuantity)
        } else {
            currentCart.removeAt(existingItemIndex)
        }
    } else if (quantity > 0.0) {
        currentCart.add(CartItem(product, quantity.coerceAtMost(maxQuantity)))
    }
    _cart.value = currentCart
}

fun processSale(kasId: String, customerId: String?, paid: Double) {
    if (kasId.isBlank()) {
        _uiState.value = SalesUiState.Error("Kas id is missing")
        return
    }
    if (_cart.value.isEmpty()) {
        _uiState.value = SalesUiState.Error("Cart is empty")
        return
    }

    viewModelScope.launch {
        _uiState.value = SalesUiState.Loading
        try {
            val items = _cart.value.map {
                val productId = it.product.id
                    ?: throw IllegalStateException("Cart contains a product without an id")
                TransactionItemInput(
                    product_id = productId,
                    quantity = it.quantity,
                    price_at_sale = it.product.price
                )
            }
            salesRepository.processSale(kasId, customerId, items, paid)
            _uiState.value = SalesUiState.Success("Sale processed successfully!")
            loadProducts()
            loadTransactions(kasId)
        } catch (e: Exception) {
            _uiState.value = SalesUiState.Error(e.message ?: "Failed to process sale")
        }
    }
}

5.3.5 Penjelasan Penerapan ViewModel Penjualan

SalesViewModel sedikit berbeda dari ViewModel lain, sebab ia memegang beberapa aliran state sekaligus. Selain _uiState untuk status proses penjualan, ada pula _cart untuk isi keranjang, _products untuk list produk, dan _transactions untuk riwayat transaksi. Masing masing memakai MutableStateFlow yang private lalu diekspos sebagai StateFlow read only, sehingga layar bisa mengamati keranjang dan list produk secara terpisah. Pemisahan ini berguna, karena keranjang sering berubah saat pengguna menambah barang, sementara status proses penjualan hanya berubah saat checkout.

Pengelolaan keranjang ditangani addToCart, dan logikanya cukup berhati hati supaya jumlah barang tidak pernah melebihi stok. Alurnya bisa diurutkan seperti berikut:
1. addToCart lebih dulu menghitung maxQuantity dari product.stock lewat coerceAtLeast, lalu nilai itu dipakai dengan coerceAtMost supaya jumlah di keranjang tidak pernah melampaui batas stok.
2. Bila produknya sudah ada di _cart, addToCart menambah quantity-nya dengan tetap dibatasi maxQuantity, lalu mengganti CartItem lama lewat copy dengan quantity yang baru.
3. Bila quantity barunya justru turun sampai nol atau kurang, addToCart mengeluarkan CartItem tersebut dari _cart.
4. Bila produknya belum ada di _cart, addToCart menambahkan CartItem baru berisi product dan quantity.
5. Di akhir, addToCart menulis list keranjang baru ke _cart.value, dan karena _cart bersifat observable, SalesScreen langsung menggambar ulang jumlah tiap barang.

Saat checkout, processSale memegang aturan yang lebih ketat, dan alurnya seperti berikut:
1. processSale lebih dulu memastikan kasId tidak kosong lewat isBlank dan _cart tidak kosong. Bila salah satunya tidak terpenuhi, processSale mengisi _uiState dengan SalesUiState.Error lalu berhenti sebelum menyentuh jaringan.
2. Setelah lolos pengecekan, processSale mengubah _uiState menjadi SalesUiState.Loading supaya tombol pembayaran bisa menampilkan indikator.
3. processSale memetakan tiap CartItem di _cart menjadi TransactionItemInput yang membawa product_id, quantity, dan price_at_sale yang diambil dari product.price.
4. processSale memanggil repository.processSale dengan kasId, customerId, list item tadi, dan paid, yang lalu mengirim p_kas_id, p_customer_id, p_items, dan p_paid ke RPC process_sale, sehingga database yang menyimpan transaksi, mengurangi stok, dan menambah saldo kas dalam satu proses.
5. Begitu berhasil, processSale mengisi _uiState dengan SalesUiState.Success lalu memanggil loadProducts dan loadTransactions.

Kedua pemanggilan ulang pada langkah terakhir itulah yang membuat stok terbaru dan riwayat transaksi ikut tersegarkan, sehingga layar yang mengamatinya akan menggambar ulang dirinya tanpa pengguna perlu memuat manual.
