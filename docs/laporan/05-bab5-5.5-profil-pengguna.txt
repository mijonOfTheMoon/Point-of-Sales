5.5 Profil Pengguna

5.5.1 Deskripsi Fitur

Fitur profil digunakan supaya tiap pengguna bisa melihat dan mengubah data akunnya sendiri. Di halaman ini ditampilkan nama, email, serta role yang sedang dipakai. Pengguna bisa membuka sebuah bottom sheet untuk memperbarui nama, email, atau password tanpa harus keluar dari aplikasi. Dari halaman ini juga tersedia tombol sign out untuk mengakhiri sesi.

5.5.4 Kode ViewModel Profil

Kotlin

fun updateProfile(newName: String?, newEmail: String?, newPassword: String?) {
    viewModelScope.launch {
        _uiState.value = AuthUiState.Loading
        try {
            val user = repository.updateProfile(newName, newEmail, newPassword)
            applyCurrentUserSnapshot(user)
            _uiState.value = AuthUiState.Success("Profile updated successfully!")
        } catch (e: Exception) {
            _uiState.value = AuthUiState.Error(e.message ?: "Update failed")
        }
    }
}

5.5.5 Penjelasan Penerapan ViewModel Profil

Pembaruan profil ditangani updateProfile pada AuthViewModel, dan function ini memakai uiState yang sama dengan login supaya layar bisa menampilkan status memuat, berhasil, atau gagal. Alur dasarnya bisa diurutkan seperti berikut:
1. updateProfile lebih dulu mengubah _uiState menjadi AuthUiState.Loading supaya tombol simpan di form bisa menampilkan indikator.
2. updateProfile memanggil repository.updateProfile dengan newName, newEmail, dan newPassword. Ketiganya bertipe nullable, jadi field yang dibiarkan kosong oleh pengguna akan dilewati, dan hanya field yang benar benar diisi yang dikirim ke Edge Function admin-users lewat action self_update.
3. Bila berhasil, updateProfile menerima objek AdminUser terbaru dari repository lalu memanggil applyCurrentUserSnapshot yang mengisi _checkState dengan AuthCheckState.Authenticated.
4. Setelah itu updateProfile mengisi _uiState dengan AuthUiState.Success supaya ProfileScreen bisa menampilkan pesan berhasil lewat snackbar.
5. Bila gagal, updateProfile menangkap exception-nya lalu mengisi _uiState dengan AuthUiState.Error berisi pesannya.

Ada satu hal penting yang terjadi di balik layar saat password ikut diganti. Pembaruan profil sebenarnya dijalankan lewat UserRepository yang memanggil Edge Function admin-users dengan action self_update. Begitu password berubah, Supabase akan menganggap kredensialnya berubah lalu membatalkan token sesi yang lama. Tanpa penanganan khusus, pengguna akan langsung terlempar keluar tepat setelah menyimpan. Untuk mencegahnya, AuthRepository akan otomatis menjalankan sign in ulang memakai kredensial baru begitu password diganti, sehingga sesinya tetap hidup dan pengguna tidak perlu login lagi.

Bagian applyCurrentUserSnapshot juga punya peran penting. Function itu langsung mengisi checkState menjadi Authenticated memakai data pengguna dari response server. Dengan begitu nama atau email terbaru bisa langsung tampil tanpa perlu membaca ulang database. Karena checkState bersifat observable, ProfileScreen yang mengamatinya lewat collectAsStateWithLifecycle akan otomatis menggambar ulang bagian identitas dengan data yang sudah diperbarui. ProfileScreen sendiri menampung perubahan ini lewat sebuah snackbar yang muncul begitu uiState menjadi Success atau Error.
